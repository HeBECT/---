❌ ОШИБКА 1: Неправильный синтаксис pattern matching
───────────────────────────────────────────────────────────
Файл: src/main.rs:604
Код: if let Ok(mut routes): Result<Vec<TradeRoute>, _> = ...

Проблема:
  • Нельзя указывать тип в pattern matching таким образом
  • Rust не поддерживает аннотацию типа после pattern

Решение:
  ✅ Изменено на: if let Ok(mut loaded_routes) = serde_json::from_str::<Vec<TradeRoute>>(&content)
  ✅ Переименована переменная routes → loaded_routes
  ✅ Тип указан в turbofish синтаксисе ::<Vec<TradeRoute>>

❌ ОШИБКА 2: Переменная routes не найдена (E0425)
───────────────────────────────────────────────────────────
Файл: src/main.rs:606, 614
Код: for route in &mut routes { ... }
      self.routes = routes;

Проблема:
  • Переменная routes конфликтовала с полем self.routes
  • Область видимости переменной была неправильной

Решение:
  ✅ Переименована локальная переменная: routes → loaded_routes
  ✅ Теперь используется: for route in &mut loaded_routes
  ✅ И: self.routes = loaded_routes

❌ ОШИБКА 3: Неправильный возвращаемый тип (E0308)
───────────────────────────────────────────────────────────
Файл: src/main.rs:18
Код: Box::new(|_cc| Ok(Box::new(KenshiTradeMap::default())))

Проблема:
  • Функция возвращала Result<Box<...>, _>
  • Ожидался Box<dyn App>
  • Лишняя обёртка в Ok()

Решение:
  ✅ Убрана обёртка Ok()
  ✅ Изменено на: Box::new(|_cc| Box::new(KenshiTradeMap::default()))
  ✅ Теперь возвращается правильный тип

⚠️ ПРЕДУПРЕЖДЕНИЕ: Неиспользуемые поля (dead_code)
───────────────────────────────────────────────────────────
Файл: src/main.rs:99-100
Поля: is_dragging, drag_start

Статус:
  ⚠️  Это только предупреждение, не ошибка
  ⚠️  Поля объявлены, но не используются в коде
  ℹ️  Можно оставить для будущего функционала
  ℹ️  Или добавить #[allow(dead_code)] если нужно

Исправление 1: Загрузка маршрутов
───────────────────────────────────────────────────────────

БЫЛО:
```rust
if let Ok(mut routes): Result<Vec<TradeRoute>, _> = serde_json::from_str(&content) {
    for route in &mut routes {
        // ...
    }
    self.routes = routes;
}
```

СТАЛО:
```rust
if let Ok(mut loaded_routes) = serde_json::from_str::<Vec<TradeRoute>>(&content) {
    for route in &mut loaded_routes {
        // ...
    }
    self.routes = loaded_routes;
}
```

Изменения:
  ✅ Убрана аннотация типа после pattern
  ✅ Тип указан через turbofish ::<Vec<TradeRoute>>
  ✅ Переименована переменная для избежания конфликта
  ✅ Код стал идиоматичным для Rust

Исправление 2: Инициализация приложения
───────────────────────────────────────────────────────────

БЫЛО:
```rust
eframe::run_native(
    "Kenshi Trade Map",
    options,
    Box::new(|_cc| Ok(Box::new(KenshiTradeMap::default()))),
)
```

СТАЛО:
```rust
eframe::run_native(
    "Kenshi Trade Map",
    options,
    Box::new(|_cc| Box::new(KenshiTradeMap::default())),
)
```

Изменения:
  ✅ Убрана лишняя обёртка Ok()
  ✅ Соответствует сигнатуре eframe::run_native
  ✅ Возвращается Box<dyn App> напрямую

Если компиляция не удаётся:
  1. Обновите Rust: rustup update
  2. Очистите кэш: cargo clean
  3. Попробуйте снова: cargo build --release

Если cargo не найден:
  1. Проверьте установку: Test-Path "$env:USERPROFILE\.cargo\bin\cargo.exe"
  2. Используйте полный путь: & "$env:USERPROFILE\.cargo\bin\cargo.exe" build --release
  3. Или используйте скрипт: .\build.ps1

Для ускорения компиляции:
  1. Используйте SSD диск
  2. Закройте другие программы
  3. Используйте cargo build (без --release) для разработки
